# 与 AI 协作开发简版模板

更新日期：2026-04-26

## 核心原则

以后开新窗口时，不需要从头讲完整项目背景。

你只需要告诉 AI 3 件事：

1. 先读哪些文档
2. 这次要做什么
3. 这次不要做什么

## 最常用模板

### 1. 通用继续开发

```text
继续这个项目。先阅读 README.md、PROJECT_FUTURE_ROADMAP.md、PRODUCT_VISION.md、PROJECT_CURRENT_STATE.md、ARCHITECTURE.md、HANDOFF.md，再查看 git status，然后继续完成：xxx。
```

### 2. 只讨论，不改代码

```text
继续这个项目。先阅读 PROJECT_FUTURE_ROADMAP.md、PRODUCT_VISION.md、PROJECT_CURRENT_STATE.md、ARCHITECTURE.md、HANDOFF.md，再查看 git status。现在不要改代码，我们讨论：xxx。
```

### 3. 只修一个问题

```text
继续这个项目。先阅读 PROJECT_CURRENT_STATE.md、ARCHITECTURE.md、HANDOFF.md，再查看 git status。然后只关注 xxx，帮我分析并修复：xxx。
```

### 4. 接着上次继续做

```text
继续这个项目。先阅读 HANDOFF.md、PROJECT_CURRENT_STATE.md，再查看 git status，然后继续完成：xxx。
```

### 5. 先分析再动手

```text
继续这个项目。先阅读 README.md、PROJECT_CURRENT_STATE.md、ARCHITECTURE.md、HANDOFF.md，再查看 git status。先不要立刻改代码，请先分析 xxx 的现状、改动点和风险，然后再开始实现。
```

## 一句话建议

以后和 AI 协作时，尽量不要让 AI 猜你的想法。

直接说清：

- 这次做什么
- 这次不做什么
- 先读哪些文档

这样就够了。
