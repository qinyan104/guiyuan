# 桌面端体验打磨

## 概述

在开始移动端适配之前，优先完善桌面端用户体验。目标让新用户注册后能立刻上手，老用户能更高效地管理数据。

## 设计原则

- **降低首次使用门槛**：新用户打开不再是空白页面
- **不引入新架构**：在现有视图/组件上做增强，不重构路由或状态管理
- **增量提交**：每个功能点独立开发，安全回退

## 四个方向

### 1. 新手引导 + 空状态

**现状：** 注册后直接跳转到 Dashboard，页面空白。族谱列表、时间线、统计数据等所有视图在首次打开时都是空的。

**改动：**

#### 1a. Dashboard 空状态
- 无族谱时显示欢迎卡片 + 快速入口
  - "创建第一个族谱"（跳转创建表单）
  - "导入族谱数据"（唤出导入对话框）
  - "浏览示例模板"（跳转族谱列表，高亮唐/明模板）
- 已有族谱时显示现有内容（不变）

#### 1b. 族谱列表空状态
- 提示文字："还没有族谱，快来创建第一个吧"
- 操作按钮：创建 + 导入

#### 1c. 时间线空状态
- 提示文字："添加人物并填写生卒年份，时间线将自动生成"
- 快速链接：跳转画布添加人物

#### 1d. 数据统计空状态
- 提示文字："数据将在添加更多人物后自动呈现"

#### 1e. 首次注册引导
- 注册成功后第一次登录，弹出一个轻量引导步骤（3 步式 modal 或 inline 引导）
  - 步骤 1: "创建或导入一个族谱"
  - 步骤 2: "在画布中添加人物"
  - 步骤 3: "填写人物详情"
- 可跳过，可通过设置重新唤起

**涉及文件：**
- `DashboardView.vue`
- `PublicationListView.vue`
- `TimelineView.vue`
- `PublicationStatsView.vue`
- 新建注册引导组件（inline 提示，非独立的引导库）

### 2. 全局搜索

**现状：** 无搜索功能。想在族谱里找人只能手动翻阅画布。

**改动：**

#### 搜索入口
- `AdminLayout` 顶栏左侧增加搜索输入框（放大镜图标）
- 聚焦时弹出搜索面板

#### 搜索范围
- 用户拥有的所有族谱中的人物名
- 族谱标题

#### 搜索结果
- 按族谱分组展示
- 点击人物 → 跳转到对应族谱的 `person-detail` 视图
- 点击族谱 → 跳转到对应族谱的 `workbench` 视图

#### 实现方案
- 后端新增 `GET /api/search?q=<keyword>` 端点
- 搜索 `persons.name`（LIKE %keyword%）+ `publications.title`
- 前端轻量 debounce（300ms），避免频繁请求
- 排序：前缀匹配（`keyword%`）优先，子串匹配（`%keyword%`）靠后

**涉及文件：**
- 后端：`SearchController.java`（新建）、`SearchService.java`（新建）
- 前端：搜索组件（`GlobalSearch.vue` 新建）嵌入 `AdminLayout.vue`
- `types/api.ts` 更新搜索响应类型
- `api/search.ts`（新建 API 封装）

### 3. 用户管理

**现状：** 用户只能注册和登录，无法修改密码、修改个人信息、设置头像。

**改动：**

#### SettingsView 增强
现有设置页面。后端已有：
- `PUT /api/user/password` — 修改密码（需旧密码验证）
- `PUT /api/user/nickname` — 修改昵称

前端做的：
- **修改昵称** — 接入现有接口
- **修改密码** — 接入现有接口（表单：当前密码 + 新密码 + 确认密码）
- **头像设置** — 上传图片作为头像（新增后端端点）
  - 后端新增 `POST /api/user/avatar` 保存头像文件
  - 前端用 `<input type="file">` 上传
  - 顶栏头像区域更新展示

**涉及文件：**
- 后端：`ProfileController.java` 扩展（新增头像端点）
- 前端：`SettingsView.vue` 重构
- `api/profile.ts` 新增方法

### 4. 数据备份 UI

**现状：** 后端 `BackupService` 已有备份/恢复能力，但无前端界面。

**改动：**

#### SettingsView 增加"数据备份"区域
- **创建备份**：按钮触发 `POST /api/backup`，生成备份文件
- **下载备份**：列出最近的备份文件，点击下载
- **恢复备份**：上传备份文件，确认后恢复

#### 现状
- `BackupService` 已在，`AdminController` 有 `GET /api/admin/backup`（需 SUPER_ADMIN 角色）
- 备份是即时下载流，不存储历史备份文件

#### 改动
- SettingsView 中增加"数据备份"区域（仅对 SUPER_ADMIN 可见）
- 直接调用现有备份端点下载

**涉及文件：**
- `SettingsView.vue` 扩展
- `api/admin.ts` 或 `api/backup.ts`（新建 API 封装）

## 不在此范围

- 移动端适配（后续再做）
- 多租户/SaaS 能力
- 性能优化（不影响现有性能即可）
- 数据库结构变更

## 测试计划

- SettingsView 组件测试（密码修改表单验证、头像上传）
- 空状态组件测试（无数据时渲染是否正确）
- SearchService / SearchController 后端测试
- 搜索前端 debounce + 结果展示测试
