# 人物传记页（详细信息）深度重构设计文档

## 1. 背景与目标
当前的人物详情页虽然具备基础功能，但在展示长篇生平（传记）时缺乏视觉吸引力和专业厚重感。
**目标**：重构为“左右分栏”布局，强化文本阅读体验，使之成为一份精美的“数字化人物传记”。

## 2. 视觉与布局设计

### 2.1 左右分栏架构 (Split Layout)
- **左侧主栏 (Main Column, 约 65-70%)**：专注“生平传记”。
  - **宣纸质感**：背景采用 `var(--bg-shell)` 配合细腻的宣纸纹理。
  - **传记排版**：增加首字放大（Drop Cap）、较大的行间距（2.0）和优化的衬线字体排版。
  - **装饰边框**：四周使用古籍风格的细线边框装饰。
- **右侧侧边栏 (Sidebar, 约 30-35%)**：专注“家族关系与元数据”。
  - **家族坐标**：显示辈分、所属宗支、血统序列。
  - **亲属磁贴**：紧凑的关系卡片，方便快速跳转。

### 2.2 核心元素美化
- **人物头图 (Hero)**：
  - **八角格头像**：头像框采用传统的八角花窗样式。
  - **情感化配色**：根据生死状态调整背景色调（在世：温润琥珀；已故：沉稳古铜/水墨）。
- **印章式标签**：身份信息（如“始祖”、“长房”）使用“朱砂印章”视觉样式。

## 3. 交互与编辑体验

### 3.1 沉浸式阅读
- **滚动同步**：左侧长文本滚动时，右侧核心信息保持悬浮（Sticky），确保上下文不丢失。
- **全屏模式**：点击可隐藏侧边栏，进入纯粹的“传记阅读模式”。

### 3.2 极简编辑
- **下划线表单**：编辑模式下，输入框取消厚重边框，仅保留优雅的下划线，模拟契约填写的真实感。
- **悬浮动作条**：保存和取消按钮始终悬浮在页面底部中央，方便长页面操作。

## 4. 技术实现细节
- **数据同步**：依托 `publication-context`，实现内存级实时同步和全局历史（Undo/Redo）支持。
- **响应式适配**：在平板和移动端自动切换为纵向堆叠布局。
- **组件化**：将关系卡片、印章标签等提取为子组件，提高复用性。

## 5. 测试验收标准
1. **长文本表现**：粘贴 1000 字以上的传记，验证排版是否舒适、美观。
2. **关系跳转**：验证从右侧亲属磁贴跳转后，页面能否瞬间切换到新人物的传记且数据准确。
3. **视觉风格**：确保配色与工作台“宣纸琥珀”主题完全融合。
