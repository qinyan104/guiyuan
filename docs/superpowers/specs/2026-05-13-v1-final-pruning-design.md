# 1.0 最终版收口设计：导出裁剪与人物详情下线

## 背景

项目已经进入 `1.0` 最终发布前的最后一轮收口阶段。当前工作台内仍存在几项不稳定或未达到可发布质量的能力：

- `单页矢量 PDF` 导出存在较多问题，当前阶段无足够时间完成修复。
- `谱书 PDF` 预览与生成链路仍需要较多打磨，不适合作为 `1.0` 正式能力对外提供。
- `分享网页` 功能表面可用，但导出后的本地 HTML 文件实际存在无法正常查看内容的问题。
- `披阅本纪（详情）` 对应的人物详情页功能思路成立，但产品完成度不足，会影响整体一致性。

本次设计的目标不是继续扩功能，而是把 `1.0` 能稳定交付的能力收干净。

## 目标

本轮只做三件事：

1. 从无涯画布中下线 `单页矢量 PDF` 与 `谱书 PDF` 两类导出能力。
2. 修复 `分享网页` 导出，确保导出的本地 HTML 文件能正常打开并查看内容。
3. 下线 `披阅本纪（详情）` 及其对应的人物详情页入口，避免保留半成品链路。

## 非目标

本轮明确不做以下事情：

- 不修复现有 PDF 导出实现。
- 不删除后端 PDF 生成服务和依赖。
- 不重做人物详情页。
- 不调整人物、族谱、分享数据结构。
- 不扩展新的导出格式或新的阅读视图。

## 方案选型

### 方案 A：只隐藏 UI

仅隐藏按钮、标签和菜单文案，保留所有路由与实现。

缺点：

- 半残能力仍可通过旧路由、旧入口或代码链路访问。
- `1.0` 发布后仍可能暴露不稳定能力。

### 方案 B：前端完整下线不稳定能力，后端暂不删除

从前端工作台、导出弹窗、页面跳转和文档层面完整移除这几项能力；底层后端实现先保留，不在本轮大拆。

优点：

- 改动集中，适合最后一轮收口。
- 对用户可见面是干净的。
- 未来恢复时可以基于现有实现重新接回。

缺点：

- 后端仍有未暴露的历史实现，需要后续有空时再做深度清理。

### 方案 C：前后端彻底删除

完全删除 PDF 导出与人物详情相关全部实现。

缺点：

- 改动面过大，不适合发布前最后阶段。
- 容易引入额外回归。

### 结论

采用 **方案 B**。

## 设计

### 1. 导出能力裁剪

前端 `导出与发布` 对话框中移除：

- `单页矢量 PDF`
- `谱书 PDF`

工作台中只保留以下导出能力：

- `分享网页`
- `矢量 SVG`
- `JSON`

同时清理与这两个 PDF 导出入口直接绑定的前端事件、方法和组件状态，避免残留不可达逻辑。

后端 `/api/publications/{id}/export/pdf` 与 `/api/publications/{id}/export/pdf/single-page` 本轮暂不删除，仅不再由前端暴露。

### 2. 分享网页导出修复

`frontend/src/features/export/shareHtmlExport.ts` 是本次修复核心。

当前判断的根因是导出模板中存在损坏字符串与模板文本异常，导致生成后的 HTML / 内嵌脚本不可靠，进而出现“文件能打开但内容无法正常查看”的结果。

本轮修复原则：

- 只修导出模板与导出结果可读性，不扩展新交互。
- 确保未加密导出和加密导出都能生成结构正确的 HTML。
- 保留当前分享网页的交互能力：
  - 族谱头部信息
  - SVG 画布展示
  - 人物卡点击详情
  - 密码保护（如设置）

实现方式：

- 修正模板中的损坏文本与错误拼接。
- 保证嵌入脚本中的字符串边界正确，不产生语法错误。
- 补一个最小测试，至少覆盖生成 HTML 后包含关键 DOM 结构与关键脚本文本，防止再次产出损坏文件。

### 3. 人物详情页下线

本轮将人物详情页视为整体下线，而不是仅移除某个按钮。

移除范围包括：

- `PersonEditorDrawer.vue` 中的 `披阅本纪（详情）` 按钮和相关事件定义。
- `WorkbenchView.vue` 中从编辑抽屉跳转到详情页的行为。
- `router/index.ts` 中 `person-detail` 路由。
- `GlobalSearch.vue` 与 `TimelineView.vue` 中跳转人物详情页的行为，改为回到当前工作台上下文中的人物定位或选中。
- `PersonDetailView.vue` 页面文件及其残留引用。

保留人物数据本身及编辑能力，不影响当前工作台中的人物编辑、关系编辑和时间线使用。

## 数据流与影响

### 导出链路

收口后导出链路应为：

- SVG：工作台 -> 前端导出器 -> 下载 `.svg`
- JSON：工作台 -> 前端导出器 -> 下载 `.json`
- 分享网页：工作台 -> `shareHtmlExport.ts` -> 下载 `.html`

不再存在用户可见的 PDF 分支。

### 人物导航链路

收口后人物查看链路应为：

- 工作台中选中人物 -> 在当前上下文中编辑与查看摘要
- 时间线 / 搜索中点击人物 -> 返回对应族谱工作台并定位或选中该人物

不再进入单独的人物详情页。

## 测试策略

至少覆盖以下验证：

1. 前端测试通过与本轮变更相关的导出/路由测试。
2. 分享网页导出后生成的 HTML 包含：
   - `#app`
   - `#tree-viewport`
   - `#detail-panel`
   - 初始化脚本
3. 本地手动验证分享网页：
   - 打开导出的 HTML 文件可看到族谱内容
   - 点击人物卡可打开详情面板
   - 设置密码时可正常解锁
4. 手动确认工作台中不再出现：
   - `单页矢量 PDF`
   - `谱书 PDF`
   - `披阅本纪（详情）`
5. 搜索、时间线与工作台中的人物入口不再跳转到独立详情页。

## 风险与缓解

### 风险 1：移除详情页后出现断链

缓解：

- 清理全部已知入口。
- 删除路由时同步搜索引用。

### 风险 2：分享网页模板修复后仍有边缘兼容问题

缓解：

- 先通过测试和 HTML 结构校验兜底。
- 再做一次实际导出打开的手动验证。

### 风险 3：文档与产品行为不一致

缓解：

- 同步更新 `README.md` 与 `docs/OPERATIONS_GUIDE.md`。
- 明确 `1.0` 当前保留的导出能力。

## 验收标准

- 用户在无涯画布中看不到两类 PDF 导出入口。
- 用户在人物编辑对话框中看不到 `披阅本纪（详情）`。
- 系统不存在可达的人物详情页路由。
- 分享网页导出后的本地 HTML 文件可正常打开并显示内容。
- 发布文档中不再把 PDF 与人物详情页描述为 `1.0` 可用能力。
