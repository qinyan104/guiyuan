# 分支挂载管理验收清单

适用版本：当前“安全挂载管理”方案。

## 验收前准备

- [ ] 已完成数据库备份
- [ ] 已准备 1 本总谱和至少 1 本分支谱
- [ ] 已准备 3 类账号：总谱 `OWNER`、分支 `OWNER`、无权限普通账号

## 权限验收

- [ ] 总谱 `OWNER` 可以进入挂载管理面板
- [ ] 非 `OWNER` 用户看不到或无法操作挂载管理入口
- [ ] 总谱 `OWNER` 在没有目标分支读取权限时，无法成功建立挂载
- [ ] 具备总谱 `OWNER` + 目标分支 `VIEWER` 权限时，可以成功建立挂载

## 挂载验收

- [ ] 普通人物可以升级为挂载点
- [ ] 建立挂载后，侧边面板能显示目标族谱名称
- [ ] 建立挂载后，主画布不会自动把远程分支混入当前可编辑数据
- [ ] 普通保存后，不会把目标分支数据误写入主族谱

## 断开验收

- [ ] 挂载点可以成功断开关联
- [ ] 断开后，人物恢复为普通节点
- [ ] 断开关联不会删除目标族谱数据

## 物理合并验收

- [ ] 挂载点可以执行物理合并
- [ ] 合并后，挂载点恢复为普通人物节点
- [ ] 合并后，总谱中可以看到新增的人物与家庭数据
- [ ] 合并后，原分支继续编辑不会自动回流到总谱
- [ ] 若保存与物理合并同时发生，系统返回 `409` 冲突提示，而不是 `500` 内部错误。

## 审计与收尾

- [ ] 审计日志里能看到建立挂载、断开关联、物理合并记录
- [ ] 合并前后都保留了备份
- [ ] 本轮验收结论已同步给各分支负责人
